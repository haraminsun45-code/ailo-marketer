# Worklog 2026-05-08

## 오늘 정한 운영 원칙

- AILO Marketer는 초기 스타트업용 저비용 반자동 콘텐츠 운영 도구로 사용한다.
- 기본 운영은 API 비용 없이 `테스트 모드`와 템플릿 생성으로 시작한다.
- 완전 자동 게시가 아니라 사람이 검토, 수정, 승인, 게시 여부를 결정한다.
- OpenAI/API/자동 게시 연동은 반복 가능한 수동 프로세스가 검증된 뒤에만 붙인다.
- 모든 대시보드 탭의 콘텐츠는 한국어로 출력한다.

## 구현된 내용

- 30일 콘텐츠 캘린더와 브리프 생성 흐름 정리.
- 현장 로그 분석 후 추천 주제를 새 브리프와 캘린더 항목으로 승격하는 흐름 추가.
- 대시보드에서 캘린더, 브리프, 초안, 품질 점수, 현장 로그 추천, 실행 로그를 볼 수 있게 구성.
- `run_daily_pipeline` 기본 동작을 초안 생성 후 사람 검토 대기로 변경.
- `--approve`를 명시해야 승인본과 dry-run 배포 확인 파일을 생성하도록 변경.
- 오래된 영어 승인본이 있더라도 최신 한글 초안을 우선 보여주도록 대시보드 미리보기 선택 로직 보강.
- OpenAI 프롬프트와 로컬 템플릿 생성 모두 한국어 출력 중심으로 정리.
- `safeText.match is not a function` 오류 수정.

## 현재 사용법

1. 대시보드 실행:

```powershell
node src/server.js
```

2. 브라우저에서 열기:

```text
http://localhost:4173
```

3. 기본 운영:

- `테스트 모드` 체크 유지
- `초안만 만들기` 체크 유지
- 콘텐츠 선택 후 `오늘 콘텐츠 실행`
- `브리프`, `블로그 초안`, `인스타그램 캡션`, `Threads`, `숏폼 스크립트` 탭 검토
- 사람이 수정/승인한 뒤에만 `초안만 만들기` 체크 해제 또는 CLI에서 `--approve` 사용

4. CLI 기본 초안 생성:

```powershell
node src/run_daily_pipeline.js --day=3 --template
```

5. 사람이 검토한 뒤 승인본/dry-run 생성:

```powershell
node src/run_daily_pipeline.js --day=3 --template --approve
```

6. 현장 로그 기반 주제 생성:

```powershell
npm run analyze:logs
npm run promote:insights
```

## 주의할 점

- `.env`에 `OPENAI_API_KEY`를 넣지 않으면 비용 없이 템플릿 모드로 운영할 수 있다.
- 기존 영어 승인본이 일부 남아 있을 수 있으므로, 앞으로는 새로 생성한 한글 초안을 기준으로 검토한다.
- 실제 API 퍼블리싱은 아직 켜지지 않았고, 현재는 dry-run 파일만 만든다.
